    Biblioteka meminfo

    Biblioteka meminfo służy do analizowania zużycia pamięci przez proces w systemie UNIX. Statystyki ukazywane są w KB, MB oraz stronach. Pozwala ona na analizę pamięci w trakcie uruchamiania procesu, alokacji, dealokacji zasobów przez użytkownia. Umożliwa ponadto obserwację ile pamięci dzielonej używa proces(obsługa pamięci dzielonej Systemu V). Można również uzyskać informacje o pamięci ze struktury mallinfo.

    Ogólny opis działania

    Struktura Memstat:

    typedef struct mem_info {

    FILE* log_file;
    long long int mmap_mem_start;
    long long int mmap_mem_usr;
    long long int mmap_addr[20];
    long long int brk_start;
    long long int brk_usr;
    long long int munmap_mem;
    long long int shm_mem;
    long long int brk_ptr;
    struct mallinfo *minfo;
    bool start;

} Mem_info;

    Biblioteka meminfo wykorzystuje strukurę Mem_info w celu zbierania informacji o pamięci w procesie. Wykonywane jest polecenie strace z nazwą analizowanego programu. Następnie wyjście strace jest zapisywane do pliku, który jest analizowany i uaktualniane są odpowiednie elementy struktury:

    • FILE *logfile - uchwyt do pliku z outputem strace
    • long long int mmap_mem_start - mapowana pamięć przy starcie procesu
    • long long int mmap_mem_usr - pamięc alokowana przez użytkownika(gdy wywoływana jest funkcja mmap)
    • long long int mmap_addr[20] - adresy zwracane z funkcji mmap
    • long long int brk_start - pamięć alokowana przy starcie procesu przy użyciu funkcji brk
    • long long int brk_usr - pamięć alokowana przez użytkownika(gdy wywoływana jest funkcja brk)
    • long long int munmap_mem - zwalniana pamięć zmapowana
    • long long int brk_ptr - wskaźnik końca stery
    • bool start - flaga mówiąca o tym czy wciąż zbieramy informacje o pamięci przy starcie procesu czy już alokowanej przez użytkownika

    Aby uzyskać informacje ze struktury mallinfo program testujący należy uruchomić z opcją -mallinfo. Program testowy musi wówczas uruchomić funckję mem_sendtoque().

    Funkcje

    mem_generate

    Mem_info *mem_generate(char *prog_name, char *option);

    Funkcja alokuje i inicjalizuje strukturę Mem_info odpowiednimi wartościami. Przyjmuje dwa argumenty: nazwę programu oraz opcję uruchomienia(w przypadku podania -mallinfo uzyskujemy informacje ze struktury mallinfo).

    mem_getstats

    void mem_getstats(Mem_info *mem_info);

    Funkcja zbiera odpowiednie informacje o pamięci do struktury Mem_info, którą dostaje jako parametr i odpowiednio modyfikuje.

    print_stats

    void print_stats(Mem_info *mem_info, char *option);

    Funkcja w odpowiednim formacie wypisuje informacje o pamięci w procesie. Przyjmuje wskaźnik na strukturę mem_info oraz opcję, która gdy jest równa -mallinfo powoduje wypisanie informacji również o strukturze mallinfo.

    mem_sendtoque

    void mem_sendtoque();

    Funkcja wykorzystywana w programie testowanym w celu przesłania informacji o strukturze mallinfo do programu testującego.

    mem_clean

    void mem_clean(Mem_info* mem_info);

    Funkcja odpowiadająca za czyszczenie zasobów.

    Zastosowane mechanizmy systemowe

    Procesy

    Funkcja mem_generate tworzy proces potomny z wykorzystaniem funkcji fork i za pomocą funkcji execlp wykonuje polecenie strace

    Wątki

    Funkcja mem_generate w przypadku podania opcji -mallinfo tworzy nowy wątek w celu stworzenia kolejki komunikatów.

    Kolejka komunikatów POSIX

    Funkcja mem_get_mallinfo tworzy kolejkę komunikatów POSIX, która jest używana do odebrania struktury mallinfo z programu testowanego.

    Uruchomienie

    Katalog zawiera pliki źródłowe biblioteki wraz z plikiem CMakeLists.txt, który umożliwia jej zbudowanie. Dostępny jest również plik testujący(mytest), który wywołuje odpowiednie funkcje biblioteczne. W celu jego uruchomienia musimy podać argument, który będzie programem testowanym(alokującym pamięć). Można również podać dodatkowy argument -mallinfo, który umożliwia uzyskanie informacji ze struktury mallinfo. Przykładowe użycie: ./mytest ./program -mallinfo. Przygotowany jest również przykładowy program do przetestowania: program.
